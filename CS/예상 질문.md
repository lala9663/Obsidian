1. 동시 요청이 많을 때 트랜잭션 충돌을 방지하는 방법은 무엇이 있을까요?
비관적 락: 데이터를 수정할 때 다른 트랜잭션이 접근하지 못하게 함
낙관적 락: 데이터를 수정할 때 버전 정보를 비교하여 충돌을 감지하고 롤백
트랜잭션 격리 수준 설정: READ COMMITTED, REPEATABLE READ, SERIALIZABLE 등 과 같은 방식이 있다.
2. 세션 기반 인증과 JWT 기반 인증 차이
세션 기반 인증: 서버가 사용자 정보를 세션에 저장, 클라이언트 세션 ID를 쿠키로 전달, 서버에서 상태 유지 필요(stateful)
JWT 기반 인증: 인증 정보를 토큰으로 발급하여 클라이언트가 저장, 서버는 토큰을 검증만 하고 별도 저장하지 않음(stateless), 확장성이 높고 마이크로서비스에서 유용
3. CORS (Cross-ORigin Resource Sharing) 란?
CORS란 도메인이 다른 서버끼리 리소스를 주고 받을 때 보안을 위해 설정된 정책이다.

2. 인덱싱을 해본적이 있나요?
(인덱싱이란 데이터베이스에서 특정 데이터를 더 빠르게 찾도록 하는 방법), 따라서 특정 데이터를 찾는데 시간을 크게 줄일 수 있다.

예상 답변:
- 경험 있을 떄: 데이터베이스에서 조회 성능을 개선하기 위해 인덱스를 사용한 경험이 있습니다. JPA를 사용할 때 @Index를 설정하거나, MySQL에서 특정 컬럼에 인덱스를 추가하여 쿼리 성능을 개선한 적이 있습니다.
- 경험 없을 때: 직접 인덱스를 설정해 본 경험은 없지만, 인덱스는 데이터 조회 속도를 높이기 위해 사용되며, B-Tree 구조를 기반으로 합니다.

추가 질문: 인덱스를 사용할 때 주의해야 할 점은?
데이터가 삽입되거나 삭제될 때마다 인덱스도 갱신해야 하기 때문에 쓰기 성능에 부하를 줄 수 있다.
또한 과도한 인덱스 생성은 오히려 성능 저하를 일으킬 수 있어, 인덱스는 필요한 컬럼에만 생성하는 것이 중요하다.

3. 옵티마이징이란 무엇인가?
옵티마이징은 시스템의 성능을 최대화하거나 리소스를 효율적으로 사용하는 작업을 의미한다.
데이터베이스나 애플리케이션 성능을 개선하려면 쿼리나 코드, 시스템 설계를 최적화하여 응답 시간을 줄이고, 자원 소모를 최소화해야 한다.
예를 들어, 데이터베이스에서 불필요한 쿼리를 최적화하거나, 서버에서 캐싱을 적용하여 처리 속도를 높이는 방법이 있다.

a. 인덱스 활용(인덱스를 추가하여 쿼리 성능 개선)
b. 쿼리 최적화(서브 쿼리 피하고, JOIN을 적절하게 활용)
c. 정규화 및 비정규화
d. 쿼리 캐싱
