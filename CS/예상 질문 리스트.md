## 기술 관련 질문

### 1. IP 주소와 서브넷 마스크
IP 주소는 네트워크 상에서 장치를 식별하는 숫자 주소이며, IPv4(예: 192.168.1.1)와 IPv6(예: 2001:db8::ff00:42:8329)로 나뉩니다. 서브넷 마스크는 IP 주소의 네트워크 부분과 호스트 부분을 구분하는 데 사용됩니다.

**추가 질문**
- 서브넷 마스크를 사용할 때 네트워크 크기를 어떻게 계산하나요?
  - CIDR 표기법(/24 등)을 사용하여 2^(32-서브넷 비트)로 계산할 수 있습니다.
- CIDR 표기법을 사용하여 IP 범위를 정의할 때 어떤 점을 주의해야 하나요?
  - 네트워크 주소와 브로드캐스트 주소를 고려해야 합니다.

---
### 2. HTTP와 HTTPS의 차이점
HTTP는 데이터를 암호화하지 않고 전송하는 프로토콜이며, HTTPS는 SSL/TLS 암호화를 사용하여 보안을 강화한 프로토콜입니다.

**추가 질문**
- HTTPS의 보안 프로토콜인 SSL/TLS에서 사용하는 암호화 방식에 대해 설명할 수 있나요?
  - 대칭/비대칭 암호화, 핸드셰이크 과정 등을 포함합니다.
- SSL 인증서의 종류(DV, OV, EV)의 차이점은 무엇인가요?
  - DV: 도메인만 인증, OV: 조직 정보 포함, EV: 확장 검증 포함

---
### 3. DNS(Domain Name System)
DNS는 도메인 이름을 IP 주소로 변환하는 시스템입니다.

**추가 질문**
- DNS 캐싱은 어떻게 작동하나요? 캐시 만료 시 문제는?
  - TTL(Time To Live) 설정에 따라 캐시된 데이터를 사용하며, 만료되면 갱신해야 합니다.
- DNS 공격의 종류에는 어떤 것이 있을까요?
  - DNS 스푸핑: 가짜 DNS 응답 전송
  - DNS 리버스 조회 공격: 도메인 정보를 통한 공격

### 4. TCP와 UDP의 차이점
TCP는 신뢰성 있는 연결을 제공하며, UDP는 빠르지만 신뢰성이 낮습니다.

**추가 질문**
- TCP의 흐름 제어와 혼잡 제어는 어떻게 동작하나요?
  - 슬라이딩 윈도우 및 AIMD(첨가 증가, 배수 감소) 방식으로 동작
- UDP가 게임 및 실시간 애플리케이션에서 많이 사용되는 이유는?
  - 지연 시간이 짧고 오버헤드가 적기 때문
